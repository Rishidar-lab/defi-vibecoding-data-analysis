# data/

All data sourced from DefiLlama's free API (no auth). Fetched 2026-06-10. Cite DefiLlama when publishing.

## Files

### arbitrum_chain_tvl_monthly.csv (seed)
Month-start Arbitrum chain TVL, downsampled from `https://api.llama.fi/v2/historicalChainTvl/Arbitrum` (daily). Excludes liquid staking and double-counted TVL per DefiLlama methodology.
- `date` — UTC month start (last row = final available daily point, 2026-05-27)
- `tvl_usd` — chain TVL in USD

### arbitrum_yields_snapshot.csv (seed)
Arbitrum pools captured from the top-TVL slice of `https://yields.llama.fi/pools`. **Coverage limited to pools ≥ ~$75M TVL** (sandbox truncation) — regenerate with the script for full coverage including GMX/Pendle/Camelot.
- `timestamp` — snapshot date (UTC)
- `asset` / `protocol` / `chain` — pool identity (`protocol` = DefiLlama project slug)
- `pool_id` — DefiLlama pool UUID (use with `/chart/{pool_id}` for history)
- `apy_pct` — current total APY, % (base + rewards)
- `apy_mean_30d_pct` — trailing 30-day mean APY, %
- `tvl_usd` — pool TVL, USD
- `sigma` — DefiLlama APY variability estimate (higher = less stable yield)
- `mu` — DefiLlama long-run mean APY estimate
- `stablecoin` — pool denominated in stablecoins
- `il_risk` — impermanent-loss exposure (`yes`/`no`)
- `exposure` — `single` or `multi` asset
- `prediction_class` / `prediction_prob_pct` — DefiLlama ML APY-direction forecast (uncalibrated third-party model; treat as a feature)
- `pool_meta` — qualifiers (e.g., lockups)

### Regenerated by scripts/fetch_dataset.py (run on open-egress machine)
- `arbitrum_chain_tvl_daily.csv` — full daily TVL history
- `arbitrum_pools_snapshot.csv` — ALL Arbitrum pools ≥ $100k TVL
- `arbitrum_pool_histories.csv` — daily APY/TVL per top pool of Vibekit protocols
- `arb_vibecoding_dataset.csv` — merged long format: `timestamp, asset, protocol, chain, apy_pct, tvl_usd, apy_vol_30d, sigma, mu, stablecoin, ilRisk, exposure, prediction_class`
